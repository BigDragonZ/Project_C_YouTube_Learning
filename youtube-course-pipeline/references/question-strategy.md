# 三轮压力测试提问策略

## 设计原则

每章笔记必须通过三轮递进式提问生成，确保从基础到高阶的完整覆盖。

## 第一轮：定义 + 推导（数学严谨性）

**目标**：建立概念的形式化基础，确保数学表达准确。

**提问模板**：
```
基于视频XX-XX的内容（章节名），请深入分析：
1) [核心概念A]的数学定义是什么？请给出标准形式。
2) [关键公式B]的完整推导过程？从假设到结论。
3) 这些定义和推导的边界条件是什么？何时失效？
请用中文回答，包含LaTeX公式。
```

**检查点**：
- 是否包含形式化定义（数学符号）
- 推导是否完整（无跳步）
- 边界条件是否明确

## 第二轮：批判 + 案例（学术深度）

**目标**：打破"教科书正确性"幻觉，建立批判性思维。

**提问模板**：
```
基于视频XX-XX的内容（章节名），继续深入分析：
1) 该理论在什么条件下失效？边界条件是什么？
2) 用具体历史案例说明理论与现实的偏差。
3) 从学术角度批判主流观点的局限性。
4) 行为经济学/信息经济学对该理论的挑战？
请用中文回答，保持学术批判深度。
```

**检查点**：
- 是否识别了理论盲区
- 案例是否具体（时间、地点、数据）
- 批判是否有文献支撑

## 第三轮：跨章节关联（知识网络）

**目标**：建立章节间的逻辑桥梁，形成知识图谱。

**提问模板**：
```
基于视频XX-XX的内容（章节名），分析：
1) 本章内容与课程其他章节的逻辑关联？
2) 本章理论在后续章节中的延伸应用？
3) 如果删除本章，整个课程的逻辑链条会如何断裂？
请用中文回答。
```

**检查点**：
- 是否识别了前置知识（ prerequisite chapters）
- 是否识别了后续深化（dependent chapters）
- 是否理解本章在课程中的结构性位置

## 执行建议

1. **每章至少 2 轮**（定义+推导、批判+案例）
2. **关键章节 3 轮**（增加跨章节关联）
3. **每轮提问后 sleep 2 秒**，避免 NotebookLM rate limit
4. **三轮结果合并**为统一模板笔记，不是分别保存

## 反模式

- ❌ 只问"请总结本章内容"（得到的是摘要，不是深挖）
- ❌ 三轮提问间隔太短（触发 rate limit）
- ❌ 不保存原始回答直接提炼（丢失细节）
- ❌ 跳过批判轮次（笔记变成教科书复述）
